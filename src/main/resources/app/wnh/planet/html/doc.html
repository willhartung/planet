Planet

Introduction

Planet, a simple planet generation application.

Planet is an application that allows the user to create a variety of simple planets with computer generated terrain. It utilizes the work of Torben Mogensen, who's website at http://hjemmesider.diku.dk/~torbenm/Planet/ appears to be down at the moment. There is a copy of the website at archive.org, the Internet Wayback Machine, at https://web.archive.org/web/20230603115922/http://hjemmesider.diku.dk/~torbenm/Planet/

The original is a model implemented by a C program and is free to use. For this application, the C program was ported, quite literally, to Java, and then an interface was built around the model to make it easier to use. The website contains a manual for the original application, as well as papers and slides about how the program works. I will not go into detail here about how it works, I can not say I fully understand it anyway.

The fundamental premise is that the generator starts with a numeric seed. With this numeric seed, it will always generate the same planet. It is a fractal generator, which will gain detail as you zoom into the renderings of the planet. The rest of the parameters to the system affect what the generated planet, and the final image, looks like.

The 10 Second Exploration

Simply, run the program and you are presented with a "Planet - Untitled" window. Click the Render button at the bottom of the window, and you will see a small planet image presented.

You can play with the Projection selections, and/or the Color Scheme values, and click the Render button again.

The Projections are used to change what kind of map is generated from the generated planet. The default Peter projection is a cylindrical projection, similar to Mercator (which is the projection most people are familiar with), but does not distort the land masses as much.

The Color Schemes are different color combinations which can try to make the planet look like a terrestrial planet, or some far off science fiction world.

Try the Print Map and Show 3D options from the Tools menu.

Main Window Controls

The main window is divided up into a parameter section on the left, and the image section on the right.

The parameter section is broken up into 5 areas.

  • Planet Generation
  • Positioning
  • Environmental
  • Presentation
  • Properties

There are also four command buttons: Export, Copy, Render, and Defaults.

Planet Generation

The parameters in this panel affect the generation and basic rendering of the planet.

The Planet Generation Parameters are as follows.

Seed - This is the numeric seed that governs the generation of the planet. It is a floating point number. It's fraction, but it can be any floating point number. Since this is the seed to an effectively random number, there's no rhyme or reason as to what seeds produce what kind of planets.

Random Seed - This button simply populates the Seed with a random number.

Lock Seed - This check box prevents the Seed from being changed. It seemed worthwhile that after finding a planet with characteristics that you like, losing the seed while playing with all of the other parameters would be unfortunate. So, you can lock the seed to prevent unintentional change.

Projection - The projection of the map used to represent the planet. If you imagine the generated planet not so much as a planet at all, but rather and crystalline ball, with rough edges etc., and then put a bright light in the middle, you can better appreciate the mechanics of what are going on here. In truth, there is no "planet" as you would think of one within the system. It's just an abstract fractal blob, and then it's project onto a map, with the results that just so happen look like a planet.

Height - The image height of the generated image.

Width - The image width of the generated image.

Color Scheme - The color scheme used to present the various altitudes of the areas on the planet.

Water % - This is a selector that allows you to generate a planet with a certain amount of water. Perhaps you want a water world, or a desert world. This is one way to affect that. Use this selector with the Search button below it.

Search - This button generates several random planets internally, until it finds one that lies within the selection Water % range. Because of the way it works, it's possible for it to find worlds just outside of the specified range. For example, if you chose 50%-59%, you may get a planet with 49% or 60%.

Actual % - This is the percentage of water that's closer to the actual coverage of the generated world. Because it uses small renderings to search for the planets, the actual percentage isn't isn't known until the large image is generated. This will only show for the PETERS projection.

Positioning

The parameters in the Positioning pane affect the position of the planet compared to the camera used for the map projection.

The Latitude and Longitude represent at what latitude and longitude of the planet is at the center of the rendered map.

The sliders mate to the nearby Lat and Long fields.

The Sun Lat and Long parameters represent where on the world the sun happens to be shining. This is used later, with the Daylight shading option.

Scale - Scale represents the magnification of the projection. Using a combination of the Latitude and Longitude values and the Scale field, you can zoom in on to specific areas of the map.

Environmental

The Environmental section has parameters to change features on the planet based on the temperature of the zone (poles being cooler).

Latitude - With this option greater than zero, the colors are changed based on the latitude of the terrain, instead of just the altitude. Simply put, the higher this number, the colder and frostier the poles will become.

Temperature - When checked, instead of seeing colors based on altitude, you will see colors based on the temperature of the land, based on latitude and altitude.

Biomes - The model has a series of colors representing different biomes. Biomes (e.g. Tundra, Grassland, Desert) are determined by a combination of rainfall and temperature. The colors below sea level for the color scheme are unchanged, but above sea level the colors are replaced with colors to represent each biome.

Base - Base is the initial altitude of the model. -0.02 is just below sea level. The most notable impact of changing this value is raising and lowering the sea level of the planet. This is another way to tune how much water is available on the planet.

Presentation

Outline - this checkbox determines whether an outline of the continents is added.

Black and White - this checkbox, combined with the Outline option, simply outlines all of the continents. Both must be checked to produce the outline.

Contours - this value determines if contour lines are shown at different altitudes. The higher the number, the more contours are added to the map.

Coastal Contours - this value determines if contours are drawn in coastal waters. The higher the number, the more contour lines that are drawn.

Latitude Lines - this value is the number of degrees between latitude lines. If the value is 0, no latitude lines are drawn.

Longitude Lines - this value is the number of degrees between longitude lines. If the value is 0, no longitude lines are drawn.

Shading - this option determines what, if any, shading is done to the map. It has 3 options: Land and Sea, Land Only, and Daylight.

When Land and Sea is selected, the contours of the map, both above and below sea level are shaded to provide a sense of the relief of the terrain. With Land Only, only the land part of the map is shaded. When Daylight is selected, the suns position (as specified in the Positioning section), is used to provide a night and day affect on the map. In this mode, the Land is also shaded just like the Land Only option.

Properties

The properties section is used to fill in any high level properties for the planet. These are only show when the Print Map option is used.

Name - the planet name.

Diameter - the planet diameter, in kilometers.

Location - a free form field for the location of the planet.

Profile - a characteristic summary of the planet, as used in some role playing games.

Command Buttons

There are four command buttons.

Export - the system prompts for a file name to save a PNG format copy of the planet image.

Copy - puts a copy of the image onto the system clipboard for pasting into other programs.

Render - renders the current image of the planet.

Defaults - resets all of the values to their defaults, including the Seed. If the Seed is Locked, it will prompt before resetting the values.

Menus

The system has three menus on the main pane. The File, Tools and Help menu.

File Menu

The file menu has options for the typical file operations, including New, Open, Save, and Save As.

The file is in an unspecified JSON format. The file is simple, but tampering with the file can lead to undefined behavior.

Tools Menu

The tools menu give access to assorted utilities for the Planet generator, including Planet Finder, Print Map, 3D View, and Color Scheme Editor.

Help Menu

The help menu gives access to a simple About dialog, and this help page.

Tools

The system has four tools accessible by the Tools menu.

  • Planet Finder
  • Print Map
  • Show 3D
  • Color Scheme Editor

Planet Finder

The Planet Finder tools makes it easier generate several planets at once in order help find the perfect planet. It has four parameters, three taken from the main window. Click the Generate button to create all of the planets. If you double-click on any of the planets in the browser, it will create a new main planet window for that planet.

Projection - This is the projection you wish to view for each planet.

Color Scheme - This is the Color Scheme desired for each planet.

Water % - This is the water range for each planet.

No. to Generate - The number of planets to generate.

Print Map

The Print Map tool takes the current planet parameters and renders it using the Icosohedral projection. On top of that, it overlays borders, a hex diagram, and then wraps it all in a frame that display the values from the Properties section of the Main Window Controls. The hex size is simply the diameter of the planet (as set in the Properties section) divided by 35, as there are 35 hexes on the equator of the map.

There is a Print button which will bring up the respective system printer dialog box. The important aspect here is that most systems allow you to print to PDF from the dialog, so its easy to generate PDF versions of your maps.

The Export button will take the map and save it to a PNG file for use in other programs.

You may also use Drag and Drop to drag the map image into another application.

Show 3D

This tool simply renders your current planet using the Peters projection, maps it onto the surface of a sphere in 3D, and starts it spinning. It's a great way to get an overall view of your planet.

Color Scheme Editor

This tool is used to create and change the Color Schemes. The program bundles several Color Schemes that were provided along with the original planet.c source code. But also allows you to change those or create new ones.

A Color Scheme consists of four functional colors, and a list of other colors that are used to color the altitudes of the generated planet. The four functional colors have pre-defined uses.

Background - This is the background color for the generated image.

Lat/Long Lines - This is the color for the Longitude and Latitude lines.

Outline 1 - When using the outline options, this is the color of the primary outline.

Outline 2 - When using the outline options, this is the color of the secondary outline. These appear when several outlines are selected, and show only on Land.

The Altitude Colors are essentially used to define gradients of color when painting the terrain. A list of values and colors are used, defining steps in the color scheme. The system will generate altitudes along the entire spectrum of the color scheme, with low values representing the various water levels (deep, shallow, etc.) and higher value representing land and mountain levels.

Sea Level is automatically assigned based on the mid point between the highest and lowest values.

Colors are interpolated between each entry. The simplest example is the Grayscale which only has two values, 5 and 255. All other colors are simply filled in between the two values. The Color Scheme can be considered a list of color gradients.

The actual values used are mostly arbitrary, altitudes will be mapped linearly among the extremes. While most of the scheme run from 0 to 255, that's not a hard limit. The system accommodates over 65000 different colors should you so choose.

Usage should be straightforward.

A Color Scheme is selected from the alphabetical list on the left, and displayed within the control grid to the right. To change an individual value, you can double-click on a cell and either update the Value directly, or, for the actual Color, you will be presented with a a Color Picker to select your color. Individual elements can be added or deleted using the + and - buttons. The elements will automatically sort based on the Value field.

To create a new Color Scheme, select the appropriate + button beneath the Color Scheme list, and enter a new name. Names can not be reused. Similarly, use the - button to delete a Color Scheme.

Color Schemes can be duplicated with the Duplicate button, and renamed with the Rename button.

The Color Schemes are automatically persisted to disk in a file located in the OS specific location for user data. If that file is removed, it will be recreated when the application restarts.
